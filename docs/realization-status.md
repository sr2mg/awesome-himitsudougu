# Realization Status

実現度は、貢献者には日本語ラベルで見せ、データ上は安定した英語enumで保存します。

## Statuses

| 表示 | 内部値 | 判断の目安 |
| --- | --- | --- |
| できた | `realized` | 市販品・実用サービスで、道具の中核機能をほぼそのまま満たしている。 |
| だいたいできた | `mostly_realized` | 見た目や制約は違うが、一般利用者の体験としてはかなり近い。 |
| 一部できた | `partial` | 中核機能の一部だけが実現している、または条件付きで近い。 |
| 研究中 | `research` | 論文、実証実験、企業R&Dなどはあるが、一般に使える製品・サービスではない。 |
| まだ | `not_yet` | 技術的な筋道は想像できるが、対応する実用品や有力な実証がない。 |
| たぶん無理 | `physically_hard` | 現行の物理法則や安全性・エネルギー制約から見て、実現見込みがかなり薄い。 |

## 対象外

魔法、寓話的効果、感情操作、一発ギャグなど、現実技術との対応を問うのが不自然なものは、実現度を選ばず `realization: null` にします。

ユーザー向けフォームでは「評価しない方がよさそう」と表示して、内部的には `evaluable: false` と `realization: null` に変換します。

## 迷いやすい境界

`だいたいできた` と `一部できた` の違い:

- `だいたいできた`: 利用者の目的はおおむね達成できる。
- `一部できた`: 近い技術はあるが、道具の中核体験までは届いていない。

`研究中` と `まだ` の違い:

- `研究中`: 具体的な論文、試作、実証実験、企業R&Dが確認できる。
- `まだ`: アイデアとしては考えられるが、根拠になる技術や実証が弱い。
